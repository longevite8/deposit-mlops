# Monitoring and Drift Detection

## Overview

Monitoring is responsible for continuously observing the health of the production system.

The monitoring subsystem tracks:

* data drift;
* prediction statistics;
* anomalies;
* retraining signals.

Its purpose is to ensure that the production model remains reliable over time.

---

# Architecture

```text
Extract
↓
Feature
├──────── Drift ─────────┐
│                        │
└────── Inference ───────┘
             │
             ▼
         Monitoring
             │
             ▼
          Alerting
```

Monitoring combines:

* drift information;
* prediction statistics;

into a unified monitoring report.

---

# Drift Detection

## Purpose

Detect whether incoming production data differs significantly from training data.

---

## Inputs

### Production Features

Generated by:

```text
Feature task
```

### Reference Dataset

Produced during training.

---

## Output

Artifact:

```python
drift_report
```

Example:

```python
{
    "drift_detected": False,
    "psi_max": 0.08,
    "drift_columns": [],
}
```

---

# PSI Thresholds

Typical interpretation:

| PSI       | Meaning           |
| --------- | ----------------- |
| < 0.1     | No drift          |
| 0.1 - 0.2 | Moderate drift    |
| > 0.2     | Significant drift |

Example:

```python
psi_max = 0.25
```

indicates severe drift.

---

# Drift Report

Example:

```python
{
```

# Monitoring Task

## Parent Tasks

```text
Inference
Drift
```

---

## Purpose

Aggregate:

* prediction metrics;
* drift metrics;

into a single report.

---

## Output

Artifact:

```python
monitoring_report
```

Example:

```python
{
    "n_predictions": 1000,
    "prediction_mean": 105.3,
    "prediction_std": 17.8,
    "drift_detected": False,
    "psi_max": 0.08,
}
```

---

# Alerting

## Purpose

Generate warnings when abnormal situations occur.

---

## Parent

```text
Monitoring
```

---

## Example Conditions

### Drift Alert

```python
psi_max > 0.2
```

---

### Empty Predictions

```python
n_predictions == 0
```

---

### Abnormal Prediction Mean

```python
prediction_mean > upper_bound
```

---

### Prediction Variance Collapse

```python
prediction_std == 0
```

---

## Output

Artifact:

```python
alert_report
```

Example:

```python
{
    "need_retraining": False,
    "severity": "normal",
}
```

Example:

```python
{
    "need_retraining": True,
    "severity": "high",
}
```

# Monitoring Philosophy

Monitoring should answer:

### Is the data changing?

Handled by:

```text
Drift task
```

---

### Are predictions behaving normally?

Handled by:

```text
Monitoring task
```

---

### Should we retrain?

Handled by:

```text
Alerting task
```

---

# Separation of Responsibilities

## Drift

Responsible for:

* feature distribution changes.

---

## Monitoring

Responsible for:

* prediction statistics.

---

## Alerting

Responsible for:

* decision making.

---

This separation keeps the codebase simple and maintainable.
